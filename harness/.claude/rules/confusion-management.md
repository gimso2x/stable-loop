# Confusion Management

불확실성과 모호함에 대처하는 규칙.

## Core Rule

**모호함을 TODO로 채우고 진행하는 것은 금지. 반드시 사용자에게 확인한다.**

## Ambiguity Protocol

1. 모호함을 감지하면 즉시 멈춘다
2. 모호한 점을 최대 3개의 구체적인 질문으로 좁혀서 사용자에게 전달
3. "어떻게 하면 좋을까요?" 같은 개방형 질문은 금지. 선택지를 제시할 것
4. 답변을 받을 때까지 다음 단계로 진행하지 않는다

## Question Format

```
X를 구현하려고 하는데, 다음 2가지가 모호합니다:
1. A에 대해 B로 처리하면 됩니까, 아니면 C로 하시겠습니까?
2. D의 기본값은 E와 F 중 어느 쪽입니까?
```

## What is NOT Ambiguity

다음은 모호함이 아니므로 질문하지 않는다:
- 표준 라이브러리 사용법
- 일반적인 코딩 컨벤션 (이미 규칙에 명시된 것)
- 명백한 버그 수정 방법
